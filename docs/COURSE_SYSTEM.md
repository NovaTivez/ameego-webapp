# Course System

## Educational Goal

The course system ensures that the application teaches communication
skills rather than only evaluating performance.

## Learning Hierarchy

Learning Track
→ Course
→ Module
→ Lesson
→ Exercise
→ Simulation
→ Feedback
→ Recommended Learning
→ Retry
→ Mastery Progress.

## Standard Lesson Structure

Every complete lesson should contain:

1. Lesson title
2. Estimated completion time
3. Learning objective
4. Short concept explanation
5. Strong example
6. Weak example
7. Explanation of the difference
8. Interactive activity
9. Knowledge check
10. Lesson summary
11. Recommended simulation

## MVP Featured Lesson

Lesson:

Building a Complete STAR Response

Learning objective:

After completing the lesson, learners should be able to structure a
behavioral interview response using Situation, Task, Action, and Result.

Lesson content:

1. Explain STAR.
2. Explain each component.
3. Show a weak answer.
4. Show an improved STAR answer.
5. Highlight differences.
6. Complete an arrangement exercise.
7. Complete a knowledge check.
8. Enter the AI interview simulator.

## MVP Exercise

Exercise type:

STAR Arrangement

The learner receives four response sections.

The learner arranges them into:

Situation
→ Task
→ Action
→ Result.

After submission:

- Show whether the order is correct.
- Explain why.
- Allow retry.
- Save completion status.

## Recommendation System

GPT-5.6 feedback should map weaknesses to lesson identifiers.

Example:

Detected weakness:

The learner explains the Situation, Task, and Action but provides no Result.

Recommended lesson:

STAR Method — Result

Recommended practice:

Retry a behavioral question with the goal of including a concrete outcome
and lesson learned.

Recommendations must be based on actual evaluation evidence.
